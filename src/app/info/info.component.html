About:

This is single page application created using Angular5 and Material design of Angular5.

In this application, we have used Angular5 routing for navigation of menus. 1. Home 2. Information

In Home component, we are displaying data in table with following features. 1. Get the data from http service, dynamically create columns, iterate data row wise and display in table. 2. Filter on whole table content. 3. Sorting on all columns. 4. Client side pagination to display 50, 100, 1000 rows on page.

In this application, I have created service for http api call and also have created pipe for capitalize first letter of text (table header), so that we can use pipe in entire application whenever required to capitalize text, no need to write logic in all components.

Assumption:

The application is built assuming that there's just one API call to get all the data and from the data, thumbnailurl are using for generating image and url is clickable which opens in a new tab.

Further possible optimisation:

The following further optimisation are possible which will significantly improve the performance,

1. Currently the pagination is being performed on client side since the API's doesn't fully support it. This means that we are storing all the data on the client side and doing all the processing on client side. The improvement could be to implement an API that returns only the requested number of items matching the filter criteria.

2. Even the filtering on data is done on the client which should ideally be done on server side to improve the performance.

3. Likewise, the sorting part can also be moved to server side.

The above optimisations will help us in keeping the client side very light and also will significantly improve the performance and responsiveness.

Limitation:

Currently, this application doesn't handle any error in-case the API fails.
